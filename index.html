Notation

Spacing utilities that apply to all breakpoints, from xs to xxl, have no breakpoint abbreviation in them. This is because those classes are applied from min-width: 0 and up, and thus are not bound by a media query.

The classes are named using the format {property}{sides}-{size}.

In FeatherCS, default values of margin and padding are not removed to ensure sufficient (or fallback) white spaces between elements. However, in some cases, they can be removed to achieve custom styling purposes. To remove margin/padding, apply the classes with this format:{property}-remove or {property}-remove-{sides}.

Where property is one of:

  • m - for classes that set margin
  • p - for classes that set padding

Where sides is one of:

  • b - for classes that set margin-bottom or padding-bottom
  • s - (start) for classes that set margin-left or padding-left in LTR, margin-right or padding-right in RTL
  • e - (end) for classes that set margin-right or padding-right in LTR, margin-left or padding-left in RTL
  • x - for classes that set both *-left and *-right
  • y - for classes that set both *-top and *-bottom
  • blank - for classes that set a margin or padding on all 4 sides of the element

Where size is one of:

  • xs - for classes that set the margin or padding to $fe-spacing-xs
  • sm - for classes that set the margin or padding to $fe-spacing-sm
  • md - for classes that set the margin or padding to $fe-spacing-md
  • lg - for classes that set the margin or padding to $fe-spacing-lg
  • xl - for classes that set the margin or padding to $fe-spacing-xl
  • xxl - for classes that set the margin or padding to $fe-spacing-xxl
