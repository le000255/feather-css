Layout

Container

FeatherCS provides a container class to wrap your content. The container class is named as fe-container. This is the default container with maximum width of 1200px.

There are helper classes helping control the size of the container at different breakpoints. These classes are named as fe-container {size}, where {size} can be xs, sm, md, lg, xl, or xxl. These classes help in making the container responsive by adjusting its width according to the screen size. For example, fe-container md will apply a medium size container that adjusts its width based on the medium breakpoint.

  • xs: Extra small (0px and up)
  • sm: Small (576px and up)
  • md: Medium (768px and up)
  • lg: Large (992px and up)
  • xl: Extra large (1200px and up)
  • xxl: Extra extra large (1400px and up)

Margin & padding

Spacing utilities that apply to all breakpoints, from xs to xxl, have no breakpoint abbreviation in them. This is because those classes are applied from min-width: 0 and up, and thus are not bound by a media query.

The classes are named using the format {property}{sides}-{size}.

In FeatherCS, default values of margin and padding are not removed to ensure sufficient (or fallback) white spaces between elements. However, in some cases, they can be removed to achieve custom styling purposes. To remove margin/padding, apply the classes with this format:{property}-remove or {property}-remove-{sides}.

Where property is one of:

  • m - for classes that set margin
  • p - for classes that set padding

Where sides is one of:

  • b - for classes that set margin-bottom or padding-bottom
  • s - (start) for classes that set margin-left or padding-left in LTR, margin-right or padding-right in RTL
  • e - (end) for classes that set margin-right or padding-right in LTR, margin-left or padding-left in RTL
  • x - for classes that set both *-left and *-right
  • y - for classes that set both *-top and *-bottom
  • blank - for classes that set a margin or padding on all 4 sides of the element

Where size is one of:

  • xs - for classes that set the margin or padding to $fe-spacing-xs
  • sm - for classes that set the margin or padding to $fe-spacing-sm
  • md - for classes that set the margin or padding to $fe-spacing-md
  • lg - for classes that set the margin or padding to $fe-spacing-lg
  • xl - for classes that set the margin or padding to $fe-spacing-xl
  • xxl - for classes that set the margin or padding to $fe-spacing-xxl

Grid layout

FeatherCS provides a 12-column grid system to help you create responsive layouts. The grid system is built using CSS grid and is fully responsive. The grid system is based on the following classes: fe-row and fe-col.

Row

The fe-row class is used to create a row in the grid system. The row class is used to create a horizontal group of columns. You can also use the gap class to add spacing between the columns.

The gap property in CSS Grid Layout defines the space between grid items and can be applied to both rows and columns. FeatherCS provides several gap sizes that you can use:

  • gap-xs - for an extra small gap
  • gap-sm - for a small gap
  • gap-md - for a medium gap
  • gap-lg - for a large gap

For example, you can use the grid-gap-md class to set a medium gap between grid items
